# Tooling stack memory

## Current stance

The repository treats personal tooling stacks as optional companion layers around a repository-native core.

## Repository-native source of truth

The following remain canonical:
- prompts
- skills
- templates
- docs
- project memory entries
- workflow artifacts committed to the repo

## Approved adoption directions

- multi-session operating model
- role-based agent lane design
- Obsidian-assisted but repo-normalized memory workflow
- optional self-hosted companion notes
- portable remote operation patterns

## Guardrails

- no paid tool should be required,
- no operating system should be required,
- no personal preference utility should become a standard,
- every external tool must have a fallback path.
