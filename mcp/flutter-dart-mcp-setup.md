# Flutter/Dart MCP Setup

## Goal
Document the expected MCP setup for Flutter/Dart-aware agent workflows.

## Recommended contents for the real repo
- required MCP server name/version
- workspace root expectations
- Flutter SDK path assumptions
- Dart/Flutter command availability checks

- MCP server starts successfully
- Flutter project root is resolved correctly
- Dart and Flutter commands can be discovered
- agent can inspect project structure without path confusion
