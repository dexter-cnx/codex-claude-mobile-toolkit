# Junie Guidelines - Flutter Toolkit

- Preserve repo conventions and architecture.
- Prefer small safe diffs.
- Keep `presentation`, `domain`, and `data` separated.
- Do not leak DTOs into UI or domain.
- Material 3 by default unless repo says otherwise.
- Respect localization workflow and avoid hard-coded UI strings.
- Add tests for meaningful new logic or bug fixes.
- Keep proposals structured: assumptions, file changes, implementation, risks, tests.
