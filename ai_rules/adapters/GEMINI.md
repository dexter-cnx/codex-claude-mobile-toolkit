# GEMINI.md - Flutter Toolkit Rules

Follow the master rules in `ai_rules/flutter_master_rules.md`.

## High-priority constraints
- Preserve architecture and folder conventions.
- Prefer explicit, deterministic code generation.
- Keep Flutter changes production-oriented.
- Avoid mixed concerns across presentation, domain, and data.

## Project defaults
- Material 3
- localization-first
- clean or lean architecture
- explicit mapping layers
- tests for meaningful behavior

## Output preference
When proposing code or changes, produce:
1. assumptions
2. changed files
3. implementation notes
4. risks
5. tests
