# Profile: Flutter Web Portfolio

Apply this profile for web-first or portfolio-style Flutter projects.

## Goals
- Excellent first-load UX
- Clear content hierarchy
- Smooth responsiveness across desktop/tablet/mobile
- Accessible interactions and keyboard-safe navigation
- SEO-aware metadata and deploy-ready static hosting flow where supported

## Rules
- Prefer lightweight sections and reusable layout primitives.
- Optimize image loading and above-the-fold rendering.
- Keep animations tasteful and not distracting.
- Avoid over-engineered architecture for mostly content-driven pages.
- Use design tokens consistently.
- Ensure external links, CV links, social links, and project cards are easy to manage from content data.

## Content workflow
If project content is data-driven:
- prefer JSON or strongly typed local content models
- separate content from widget structure
- keep theme tokens and content models independent
