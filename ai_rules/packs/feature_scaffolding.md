# Task Pack: Feature Scaffolding

Use this pack when creating a new feature.

## Workflow
1. Infer current architecture and feature shape.
2. Propose the exact file tree first.
3. Create only the files required for the first usable slice.
4. Avoid generating empty placeholders unless the repo convention demands them.
5. Include test seams from the start.

## Expected output
- assumptions
- proposed file tree
- rationale for each file
- implementation notes
- initial test plan

## Rules
- Prefer vertical feature slices.
- Keep the first iteration small but complete.
- Respect state-management conventions already present in the repo.
