# Markdown Workspace Capability Adoption

This document explains how to extract reusable architecture value from a markdown workspace or Obsidian-like Flutter app without turning the toolkit into a note-app-only repository.

## Extract into core toolkit patterns
- offline-first repository pattern
- local SQLite FTS search pattern
- optional local-first sync guidance
- export/backup architecture guidance
- secure local data guidance

## Keep optional or reference-only
- wiki-link parsing
- backlinks
- graph view
- Obsidian-like shell layouts
- note-specific side panels
