# Mobile Design Direction Lane

Use this lane when a feature is too vague to implement directly from rough product notes.

## When to run

Run before implementation when you need:
- a distinct visual direction
- a better component hierarchy
- token decisions before coding
- a Stitch-ready handoff
- a stronger critique pass for an existing screen

## Core sequence

1. Capture a concise feature purpose
2. Write a visual direction brief
3. Define tone, density, contrast, and interaction posture
4. Convert the direction into tokens and component rules
5. Produce screen guidance for implementation

## Recommended assets

- `skills/flutter-art-direction/`
- `prompts/design/mobile_ui_art_direction.md`
- `templates/ux/mobile_theme_brief.template.md`

## Output expectations

The result should be implementation-ready, not just inspirational:
- visual direction
- component hierarchy
- token guidance
- accessibility considerations
- risky areas called out
