# Adoption Playbook

## Goal

Help a team adopt this toolkit without copying everything blindly.

## Recommended rollout

### Phase 1 - Foundation

Adopt only:

- `AGENTS.md`
- one agent-surface doc relevant to your runtime
- `docs/skill-workflow-map.md`
- `prompts/plan_feature.md`
- `prompts/verification_pass.md`

### Phase 2 - Workflow discipline

Add:

- `docs/verification-workflow.md`
- `docs/project-memory-system.md`
- `project_memory/`
- `templates/project-memory-update-report.md`

### Phase 3 - Multi-surface maturity

Add:

- `docs/runtime-matrix.md`
- `docs/agent-surface-contract.md`
- Antigravity or OpenClaw-specific docs only if needed

## Common adoption mistakes

- importing every file at once
- using skills without stable rules
- skipping verification artifacts
- leaving project memory stale
- adding new prompts before checking existing routing docs
