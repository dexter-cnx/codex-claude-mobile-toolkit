# Accessibility Review Checklist

## Semantics
- [ ] Icon-only actions have labels
- [ ] Decorative elements are not noisy
- [ ] Key content is announced meaningfully

## Text and layout
- [ ] Larger text does not break core flow
- [ ] Important controls remain visible
- [ ] Essential copy is not truncated beyond usability

## Interaction
- [ ] Primary actions are obvious
- [ ] Touch targets are reasonable
- [ ] Focus/navigation is sensible when relevant

## State communication
- [ ] Loading state is understandable
- [ ] Empty state is understandable
- [ ] Error state offers recovery
- [ ] Success feedback is noticeable
