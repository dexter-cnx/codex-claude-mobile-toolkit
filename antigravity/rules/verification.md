# Verification Rules

## Principle
No Antigravity task is complete without an explicit verification section.

## Static checks
Minimum expected checks:
- dependency resolution
- formatting
- static analysis
- automated tests relevant to the change

## Runtime checks
When applicable:
- launch the app on a target device or simulator
- exercise the changed screen or flow
- capture screenshots or notes

## Reporting
The verification report must state:
- which commands were run
- whether they passed or failed
- what could not be verified
- any follow-up required
