# Workflow: REST Catalog Feature Delivery

## Goal
Deliver a Flutter feature backed by a read-mostly REST API using a predictable, verification-first workflow.

## Stages
1. inspect existing repository conventions
2. define feature scope and states
3. inspect or define remote and local data paths
4. implement presentation/domain/data changes
5. verify
6. emit artifact report

## Required outputs
- changed files
- cache behavior summary
- verification results
- known gaps
