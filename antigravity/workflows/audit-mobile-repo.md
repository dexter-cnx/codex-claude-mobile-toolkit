# Audit Mobile Repo

## Objective
Produce a structured health audit for a Flutter/mobile repository.

## Phases
1. inspect repository shape
2. review architecture and state boundaries
3. review localization and testing posture
4. review platform and tooling health
5. identify drift and stale patterns
6. generate audit artifact
7. recommend next workflows

## Required artifacts
- audit report
- prioritized action list

## Suggested supporting skills
- `flutter-audit-repo`
- `flutter-repo-conventions`
- `flutter-simple-clean-architecture-guard`
- `flutter-review-localization`
- `flutter-review-testing`
- `flutter-review-platform-config`
