# Finalize Mobile Change

## Objective
Move a completed Flutter/mobile change into a review-ready state with explicit artifacts.

## Phases
1. scope summary
2. impact classification
3. baseline verification
4. focused review passes
5. artifact generation
6. readiness declaration

## Required artifacts
- verification report
- change summary
- risk log if needed

## Suggested supporting skills
- `flutter-finalize-change`
- `flutter-review-architecture`
- `flutter-review-localization`
- `flutter-review-testing`
- `flutter-review-platform-config`
- `flutter-pr-reviewer`
