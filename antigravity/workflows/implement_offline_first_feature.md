# Implement Offline-First Feature

## Goal

Deliver a Flutter feature that remains useful offline and follows local-first repository design.

## Steps

1. read root `AGENTS.md`
2. read `prompts/new_feature.md`
3. read `prompts/patterns/offline_first_repository.md`
4. inspect the target project architecture
5. define local store and repository boundaries
6. define state model including stale/syncing/error
7. implement feature changes
8. verify local behavior without network assumptions
9. emit verification and risk artifacts
