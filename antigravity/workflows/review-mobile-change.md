# Review Mobile Change

## Objective
Produce a structured quality review for a Flutter/mobile change.

## Phases
1. review change intent and scope
2. inspect changed files
3. assess architecture fit
4. assess UX, localization, and testing posture
5. assess platform risks
6. generate concise review findings

## Required artifacts
- review summary
- follow-up list
- verification expectation notes

## Suggested supporting skills
- `flutter-review-change`
- `flutter-pr-reviewer`
- `flutter-review-architecture`
- `flutter-review-localization`
- `flutter-review-testing`
- `flutter-review-platform-config`
