# Purpose

Use this skill when creating a new Flutter app or restructuring a small-to-medium codebase into a maintainable baseline.

# Outcomes

- clean layer boundaries
- thin presentation widgets
- networking outside UI
- CSV-first localization flow
- one sample feature slice

# Rules

1. No business logic in widgets.
2. No API calls in presentation.
3. Keep domain framework-light.
4. Keep data responsible for DTOs and mapping.
5. Prefer practical abstractions.

# Process

1. Read the references.
2. Propose the folder structure.
3. Create bootstrap files.
4. Add one sample feature.
5. Run a coherence pass.
