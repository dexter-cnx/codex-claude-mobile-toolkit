# flutter-self-improve

## Purpose

Capture reusable lessons from completed work and route them into the correct repository assets.

## Use when

- the same correction appeared multiple times,
- a pipeline or skill felt incomplete,
- a report format was insufficient,
- duplication or naming drift was discovered.

## Procedure

1. extract the reusable lesson,
2. decide if it is durable enough to keep,
3. pick the smallest appropriate destination,
4. prefer editing an existing asset,
5. create a new asset only when needed,
6. record the improvement log.

## Routing destinations

- `AGENTS.md`
- `CLAUDE.md`
- `GROK.md`
- `prompts/`
- `skills/`
- `templates/`
- `docs/`
- `antigravity/rules/`
- `antigravity/workflows/`

## Expected outputs

- `templates/improvement-log.md`
- destination decision
- brief rationale
- note on whether Antigravity needs mirroring

## Verification

Every self-improvement proposal should be tied to a concrete repeated issue, missing step, or reusable lesson.
