# flutter-commit-readiness

## Purpose

Decide whether a change is ready to be committed as a coherent unit.

## Use when

- implementation may be done,
- verification exists but readiness is unclear,
- the task should be split into one or more commits.

## Review focus

- scope coherence
- obvious leftovers
- verification sufficiency
- whether follow-ups should be separated from the current commit

## Expected outputs

- readiness statement
- suggested commit theme
- split recommendations if the scope is too broad
