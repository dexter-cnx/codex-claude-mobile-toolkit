# flutter-review-architecture

## Purpose

Review whether a change respects the toolkit's pragmatic clean architecture boundaries.

## Review focus

- presentation vs domain vs data separation
- business logic leakage into widgets
- repository and transport boundaries
- proportionate layering
- readability and maintainability

## Expected outputs

- architecture findings
- concrete risks
- recommended adjustments

## Verification

Ground findings in the changed files and surrounding module structure.
