# flutter-review-testing

## Purpose

Review whether the testing posture matches the risk and scope of the change.

## Review focus

- existing tests touched or affected
- missing regression coverage
- mismatch between claimed verification and actual evidence
- whether the change needs widget, unit, integration, or smoke coverage

## Expected outputs

- testing findings
- evidence quality assessment
- highest-value missing checks

## Verification

Distinguish clearly between:
- checks already run,
- checks recommended,
- checks not possible in the current environment.
