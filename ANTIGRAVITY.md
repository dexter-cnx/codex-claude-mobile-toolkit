# ANTIGRAVITY.md

This repository is structured so Anti Gravity can route work through reusable skills.

## Recommended usage
- use prompt files in `prompts/` as task entry points
- use `skills/` as the canonical workflow definitions
- use `core/` as invariant engineering guidance

## Good task bindings
- project bootstrap -> `prompts/new_project.md`
- feature work -> `prompts/new_feature.md`
- audit existing app -> `prompts/audit_existing_project.md`
- ui polish or stitch handoff -> `prompts/improve_ui_with_stitch.md`
- csv localization rollout -> `prompts/localize_from_csv.md`
- performance pass -> `prompts/optimize_performance.md`

## Anti Gravity expectations
Outputs should be repository-friendly. Prefer folder trees, file plans, acceptance checklists, and patch-ready markdown over abstract guidance.
