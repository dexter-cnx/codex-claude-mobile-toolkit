# Implement Nothing-inspired Flutter screen prompt

Plan or implement a Flutter screen using the optional Nothing-inspired overlay from the toolkit.

## Required inputs

- screen goal,
- target user,
- primary action,
- key states,
- whether the style is mandatory or exploratory,
- dark mode and light mode expectations.

## Tasks

1. decide whether the style is appropriate for this screen,
2. define primary, secondary, and tertiary hierarchy,
3. choose one expressive moment only,
4. map the screen to reusable widgets before writing layout code,
5. rely on semantic tokens and theme hooks,
6. preserve accessibility and localization resilience,
7. keep the implementation practical for long-term Flutter maintenance.

## Output

Produce:

- fit verdict,
- hierarchy map,
- widget inventory,
- token usage notes,
- implementation plan,
- code structure guidance,
- risk notes for accessibility, text scaling, and localization,
- verification checklist.

## Constraints

- avoid hardcoded color values in leaf widgets,
- avoid shadow-heavy or effect-heavy styling,
- keep touch targets appropriate,
- always pair stylized data visuals with exact values,
- do not imitate the reference look so literally that usability regresses.
