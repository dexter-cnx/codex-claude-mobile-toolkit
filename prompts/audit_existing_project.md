# Prompt: Audit Existing Project

Use `flutter-existing-project-audit` as the primary skill.

## Goal
Review an existing Flutter codebase and produce a practical audit that separates findings, severity, recommendations, and refactor steps.

## Expected output
- assumptions and project context
- architecture findings
- state-management findings
- localization findings
- performance risks
- testing gaps
- prioritized refactor plan
- quick wins vs deep changes

## Constraints to honor
- do not propose a rewrite unless the evidence is overwhelming
- distinguish symptoms from root causes
- prefer incremental migration plans
