# Self Improve Toolkit

## Goal

Extract reusable lessons from completed work and route them into the right permanent repository assets.

## Use when

- repeated guidance appeared during a task,
- a workflow step was missing or weak,
- a skill became too broad,
- an output format proved insufficient,
- a new recurring pattern emerged.

## Inputs

- completed task summary
- repeated corrections or clarifications
- files or workflows that felt missing, vague, or duplicated
- any artifact produced during the task

## Inspect first

- `AGENTS.md`
- agent-surface files
- relevant pipeline prompt
- related skills
- related templates
- related docs
- Antigravity mirror files if the workflow has one

## Routing flow

### 1. Identify the lesson
Write the reusable lesson in one or two sentences.

### 2. Decide whether it is durable
If it is one-off or hyper-specific, do not create a new asset.

### 3. Pick the smallest appropriate home
Use this priority order:

- rules
- pipeline prompt
- skill
- template
- docs
- Antigravity mirror

### 4. Prefer updating existing assets
Only create new files when the pattern is clearly reusable and not already covered.

### 5. Produce an improvement log
Use `templates/improvement-log.md`.

## Required outputs

- lesson summary
- destination file
- why that destination is correct
- proposed change summary
- whether Antigravity also needs an update
