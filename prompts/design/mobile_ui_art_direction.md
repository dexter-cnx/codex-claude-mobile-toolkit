# Prompt: Mobile UI Art Direction

Use `flutter-art-direction`.

## Goal

Turn a rough feature or screen idea into an implementation-ready visual direction for a Flutter/mobile product.

## Inputs to provide

- feature or screen name
- user goal
- product context
- audience
- current pain points
- constraints
- existing design-system notes if any

## Requested output

Produce:

1. design intent
2. visual direction with a clear aesthetic stance
3. design-token recommendations
4. component-level guidance
5. motion and interaction guidance
6. accessibility considerations
7. Flutter implementation notes
8. optional Google Stitch handoff notes

## Hard requirements

- avoid generic, trend-chasing UI language
- make the direction buildable in Flutter
- tie every aesthetic claim to an implementation implication
- account for light and dark mode
- call out where Material 3 should be followed strictly versus selectively customized
