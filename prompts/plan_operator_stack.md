# Plan operator stack

Use this prompt when the repository needs a concrete operating model for multi-agent or multi-session work.

## Task

Design an operator stack plan for this repository.

## Deliverables

Produce:
1. a concise plan,
2. a proposed file change list,
3. a role topology,
4. a session topology,
5. a migration path from current state,
6. a risk list,
7. a recommended first rollout.

## Constraints

- Do not require paid tools.
- Do not require one operating system.
- Preserve compatibility with Codex, Claude Code, Antigravity, and OpenClaw surfaces already present in the toolkit.
- Keep the repository as the source of truth.
- Treat external tools as optional adapters.

## Evaluation criteria

A good plan:
- simplifies operation,
- reduces context loss,
- improves memory quality,
- improves review quality,
- remains portable.

## Preferred output format

Use headings, tables only when necessary, and commit-ready markdown.
