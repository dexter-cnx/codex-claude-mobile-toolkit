# Prompt: Error Handling Pass

Review the change for resilience and user-facing failure handling.

Check:
- loading state
- success state
- empty state
- recoverable failure state
- retry path
- timeout / offline-ish behavior
- telemetry hook points
- copy quality

Use:
- `docs/repo_discipline/error-handling-standard.md`
- `templates/checklists/error-handling-review.md`

Output:
- Scope Reviewed
- State Coverage Matrix
- Failure Gaps
- Recommended Fixes
- Release Risk Note
