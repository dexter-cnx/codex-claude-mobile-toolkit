# Improve existing UI prompt

Review an existing Flutter screen or flow and propose concrete UI/UX improvements that preserve maintainability.

## Inspect first
- current screen widgets
- theme and token files
- reusable components
- localization usage
- existing state handling
- forms, lists, dialogs, and navigation patterns

## Required output
Produce:
1. current-state critique
2. UX issues ranked by severity
3. component changes recommended
4. spacing and hierarchy improvements
5. accessibility concerns
6. localization and overflow concerns
7. low-risk implementation plan
8. verification notes

## Constraints
- do not recommend changes that require a complete design-system rewrite unless necessary
- prefer incremental improvements that fit the repo
