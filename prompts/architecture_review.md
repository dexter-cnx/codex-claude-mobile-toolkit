# Architecture Review

Use this prompt when a change may affect repo boundaries, layering, or design consistency.

## Required behavior

Review the target implementation or plan and evaluate:

1. Which architectural layers are involved?
2. Whether responsibilities are placed in the correct layer.
3. Whether any boundary is being crossed for convenience.
4. Whether the proposed design increases or reduces coupling.
5. Whether naming, ownership, and flow are consistent with the repo's existing model.
6. Which tradeoffs are acceptable and which are not.

## Output format

### 1. Architectural context
### 2. Layers involved
### 3. Boundary compliance review
### 4. Coupling and maintainability review
### 5. Risks introduced
### 6. Recommended adjustments
### 7. Verdict
