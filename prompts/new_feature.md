# Prompt: New Feature

Use `flutter-feature-scaffold` as the primary skill.

## Goal
Add a new feature to an existing Flutter app with maintainable boundaries and clear implementation steps.

## Expected output
- feature boundary definition
- file tree for the feature
- state objects and actions/events
- route changes
- repository and datasource needs
- localization keys
- acceptance checklist

## Constraints to honor
- no business logic in widgets
- no premature abstraction
- keep feature naming consistent with the repo
